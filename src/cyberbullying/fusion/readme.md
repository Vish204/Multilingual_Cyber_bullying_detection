## Overview

This module implements the fusion layer of the cyberbullying detection system. Its main goal is to combine multiple model predictions to generate a more accurate and robust final decision.

The fusion integrates outputs from three components:

MTKD Student Model → Predicts cyberbullying probability (p_cb)

Sarcasm Model → Predicts sarcasm probability (p_sarcasm)

Emotion Model → Predicts emotion intensity (p_emotion)

By combining these probabilities with a weighted fusion formula, the system can account for subtle cues such as sarcasm or emotional tone that might mask abusive language.

## Fusion Logic

The fusion formula used in this module is:

fusion_score = 0.60 * p_cb + 0.25 * p_sarcasm + 0.15 * p_emotion

fusion_score is a value between 0 and 1.

The final binary prediction is computed using a threshold of 0.5:

>= 0.5 → Bullying

< 0.5 → Non-bullying

## Inputs

The fusion module operates on probability outputs generated by the individual models. These probabilities are CSV files stored in:

notebooks/analysis_results/fusion/
File	Description
mtkd_probs.csv	Cyberbullying probabilities from the MTKD XGBoost model (p_cb)
sarcasm_probs.csv	Sarcasm probabilities from the sarcasm model (p_sarcasm)
emotion_probs.csv	Emotion probabilities from the emotion model (p_emotion)

These files are produced by the respective prediction scripts in this folder.

## Outputs

After running the fusion process, the module produces:

File	Description
fusion_predictions.csv	Combined fusion scores and final binary predictions
fusion_metrics.json	Evaluation metrics (accuracy, F1, precision, recall, ROC-AUC)
fusion_confusion_matrix.png	Confusion matrix visualization
fusion_roc_curve.png	ROC curve visualization

## Purpose and Role in the System

The fusion module is not a model on its own, but a decision-making layer that consolidates signals from multiple sources:

Cyberbullying detection (MTKD) → Primary indicator.

Sarcasm detection → Adjusts for masked abuse.

Emotion detection → Provides contextual intensity of the text.

The fusion module ensures more reliable and robust predictions, which are then used in:

Offline evaluation pipelines (Phase 3.5)

Real-time system testing (Phase 3.8)

Final system integration in Phase 4

## Workflow

Run each component prediction script to generate probability CSVs.

Run fusion.py to compute fusion scores and generate final predictions.

Use compute_fusion_metrics.py to evaluate the fusion output and produce visualization/metrics.

The results are stored in notebooks/analysis_results/fusion/ for later inspection or system integration.

## Notes

The fusion module assumes all input CSVs have the same sample order.

Probabilities are combined using predefined weights, which can be adjusted based on experimentation.

This module allows rapid evaluation of system-level performance without retraining any models.
