# Fusion Module

## Overview

This module implements the **fusion layer** of the cyberbullying detection system. Its primary objective is to combine predictions from multiple models to produce a **more accurate and robust final decision**.

The fusion mechanism integrates outputs from three components:

- **Student_V2 Model (MTKD)** → Cyberbullying probability (`p_cb`)
- **Sarcasm Detection Model** → Sarcasm probability (`p_sarcasm`)
- **Emotion Detection Model** → Emotion intensity probability (`p_emotion`)

By combining these probabilities using a **weighted fusion strategy**, the system can capture subtle linguistic signals such as sarcasm or emotional tone that may conceal abusive language.

---

## Fusion Logic

The fusion score is computed using the following weighted formula:

```

fusion_score = 0.60 * p_cb + 0.25 * p_sarcasm + 0.15 * p_emotion

```

Where:

- **p_cb** → Cyberbullying probability from the Student_V2 model  
- **p_sarcasm** → Sarcasm probability from the sarcasm model  
- **p_emotion** → Emotion intensity probability from the emotion model  

The fusion score ranges between **0 and 1**.

The final binary classification is determined using a threshold:

```

fusion_score ≥ 0.5  → Cyberbullying
fusion_score < 0.5  → Non-cyberbullying

```

---

## Inputs

The fusion module operates on probability outputs generated by the individual models. These probability files are stored in:

```

notebooks/analysis_results/fusion/

```

| File | Description |

## Purpose and Role in the System

The fusion module acts as a **decision-level integration layer** rather than a standalone model.

It consolidates signals from three specialized subsystems:

1. **Cyberbullying Detection (Student_V2 / MTKD)**  
   - Primary indicator of abusive language.

2. **Sarcasm Detection**  
   - Helps detect hidden or sarcastic bullying.

3. **Emotion Detection**  
   - Provides contextual emotional intensity of the text.

By combining these signals, the system achieves **more reliable and context-aware cyberbullying detection**.

This module is used in:

- Offline evaluation pipelines  
- System performance analysis  
- Final prediction generation for inference pipelines  

---

## Workflow

The fusion pipeline follows these steps:

1. Run each component prediction script to generate probability outputs:

```

predict_mtkd_prob.py
predict_sarcasm_prob.py
predict_emotion_prob.py

```

2. Execute the fusion step:

```

fusion.py

```

This computes fusion scores and generates final predictions.

3. Evaluate the fusion output:

```

compute_fusion_metrics.py

```

This produces evaluation metrics and visualizations.

4. Perform error analysis:

```

run_error_analysis.py
plot_error_analysis.py

```

All generated outputs are stored in:

```

notebooks/analysis_results/fusion/

```

---

## Notes

- The fusion module assumes **all input CSV files maintain the same sample order**.
- Fusion weights (`0.60`, `0.25`, `0.15`) were selected based on experimental validation.
- This module allows **system-level performance evaluation without retraining individual models**.
- The architecture can be easily extended to include additional signals such as toxicity classifiers or contextual embeddings.
